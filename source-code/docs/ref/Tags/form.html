A plugin for creating a Vaadin application in Grails.

(Quick Reference)

form

Purpose

Adds a Vaadin Form component to the parent container.

Examples

<v:form bean="${bookInstance}" writeThrough="false">
  <v:field name="title" readOnly="false" required="true" requiredError="Please enter a Title">
    Title
  </v:field>
  <v:text name="author" readOnly="false" required="true" requiredError="Please enter an Author">
    Author
  </v:text>
  <v:date name="datePublished" readOnly="false" required="false">
    Date Published
  </v:date>
  <v:textArea name="description" readOnly="false" required="false"/>
    Description
  </v:textArea>
</v:form>

Description

The Form component is configured using nested field tags. These are used to configure the formFieldFactory and set the visibleItemProperties properties of the form component.

Body

  • field tags should be specified in the body. Any HTML is ignored.

Attributes

Specific
  • bean (optional) The bean (e.g. domain instance) to use to provide the underlying form data.
  • itemDataSource (optional) The Item object to use to provide the underlying form data.

Vaadin's BeanItem class cannot handle embedded or nested properties.

This plugin provides a DomainItem class that supports embedded and nested properties. The bean attribute automatically uses this class.

General
  • var (optional) Uses Grails's set tag to set the value of the specified variable to be this component.
  • class (optional) The CSS class name to add to the component using the addStyleName() method
  • sizeUndefined (optional) Calls setSizeUndefined() on the component. (Attribute value is ignored)
  • sizeFull (optional) Calls setSizeFull() on the component. (Attribute value is ignored)
  • location (optional) The location slot in the parent container to which this component should be added.
  • wrapperClass, wrapperStyle (optional) Special styling for wrapper <div>.

See Vaadin Form API docs for full list of allowed attributes.
